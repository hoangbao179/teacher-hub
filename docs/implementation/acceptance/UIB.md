# UIB Acceptance

- [x] Mobile uses a single-column full-width shell with compact padding and a
  visible, accessible bottom navigation.
- [x] Desktop hides bottom navigation, shows coherent side navigation and uses
  approximately 960–1200 px of useful content width.
- [x] Sticky actions do not overlap mobile navigation and forms remain bounded on
  desktop.
- [x] Dashboard, class, student, lesson, tuition, reconciliation, calendar,
  busy-slot, homepage and login screens follow the approved hierarchy and density
  direction.
- [x] There is no page-level horizontal overflow at 360 px.
- [x] Automated responsive tests cover both navigation variants, width usage,
  bounded forms, sticky actions and core actions.
- [x] Required before/after screenshots and `UIB-visual-review.md` exist and every
  primary route is manually reviewed at all required viewports.
- [x] `check:fast`, E2E and build gates pass.
