# M1.1 Implementation Report

## Initial repository state

- Branch `dev`, tracking `origin/dev`.
- Không có tracked modification hoặc staged diff khi bắt đầu.
- Có ba workbook legacy untracked trong `docs/reference/legacy-excel/`; chúng
  được coi là dữ liệu có sẵn của user và giữ nguyên.
- Repo chỉ có một unit-test file, OpenAPI thiếu routes, status phân tán giữa
  `BASE_STATUS.md`/milestones và `FILE_MANIFEST.txt` được duy trì thủ công.

## Problems found

- Client tin `teacher-user` trong localStorage và protected route redirect trước
  khi token được backend kiểm tra; xử lý 401 không tập trung.
- Service cho phép giá lớp bằng 0; paused class vẫn nhận enrollment.
- Rule one-active-enrollment chỉ dựa vào `SELECT` trước `INSERT`, có race.
- `audit_logs` tồn tại nhưng mutations M1 không ghi audit.
- OpenAPI chỉ mô tả một phần endpoints và thiếu update/state/schedule routes.
- Nhiều control nhìn như có thể bấm nhưng không có handler (Home contact,
  makeup/unrecorded, mark-paid); internal navigation còn dùng raw `href`.
- Thiếu native-MySQL integration runner, browser smoke, seed/reset và consistency check.

## Architecture changes

- Chuẩn hóa priority: business rules/approved ADR → acceptance → shared contracts
  → OpenAPI → source/schema → wireframe.
- `docs/implementation/status.md` là status source duy nhất; `BASE_STATUS.md` chỉ
  còn con trỏ và manual `FILE_MANIFEST.txt` đã xóa.
- Thêm reusable `AuditRepository`; controller truyền actor, repository ghi audit
  cùng transaction với business mutation.
- Thêm CRUD endpoint nền cho recurring schedules với service validation,
  repository SQL và audit đúng layering.

## Documentation changes

- Thêm roadmap, task, acceptance và status M1.1; đồng bộ class/student feature,
  database/frontend architecture, README và verification workflow.
- Đồng bộ OpenAPI với đủ 33 Express method/path thực tế.
- Thêm bốn ADR approved: participant snapshot, effective-dated tuition policy,
  historical enrollment eligibility và late-entry ordering.

## Database migrations

- `0003_m1_1_integrity.sql`:
  - `CHECK (classes.default_package_price > 0)`;
  - generated `class_enrollments.active_student_key`;
  - unique `uq_enrollments_one_active_per_student` làm consistency boundary.
- Migration đã áp dụng thành công trên native schema `teacher_hub_test`; không
  reset hoặc migrate database ứng dụng `teacher_hub` trong task.

## API changes

- Thêm `POST /api/auth/logout`.
- `GET /api/auth/me` đọc authoritative user từ database và từ chối user disabled.
- Thêm `POST /api/classes/:id/schedules`, `PATCH/DELETE
  /api/recurring-schedules/:id`.
- Class/student/enrollment mutations truyền actor cho audit.
- Không thêm API lesson-recording/M2 mới.

## Authentication changes

- Boot đọc token, gọi `/api/auth/me`, có `bootstrapping` state và không lưu/tin
  stored user JSON.
- Protected route chờ bootstrap, redirect `/admin/login` và giữ intended path.
- API client luôn attach token, xử lý mọi 401 bằng xóa token + notify auth context.
- Token hợp lệ giữ session khi refresh; token hết hạn/invalid logout; Home vẫn public.

## UI stabilization

- Thêm pause/resume enrollment và logout control.

- M2/M4/unconfigured actions bị disable và ghi rõ chưa triển khai/cấu hình.
- Home chỉ hiện admin-login link trong `import.meta.env.DEV`.
